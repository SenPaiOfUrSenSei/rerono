# Rerono

**Rerono** is a modern, cross-platform (Windows & Linux) command-line URL and domain blocker designed for productivity. It allows you to block entire websites or specific pages/paths (like `youtube.com/shorts`) for set periods, helping you stay focused and eliminate distractions.

Unlike traditional hosts-file blockers which can only block entire domains, Rerono uses a local Man-in-the-Middle (MITM) proxy server to inspect and filter HTTPS requests at the URL path level.

---

## Key Features

- 🌐 **Granular Path Blocking:** Block specific URLs (like `youtube.com/shorts` or `facebook.com/watch`) while keeping the rest of the site functional.
- 💻 **Cross-Platform:** Works seamlessly on both Windows and Linux.
- ⚡ **Rust-powered Virtual Environments:** Automatically runs and isolation-manages `mitmproxy` via the state-of-the-art `uv` python package runner.
- 🛡️ **Zero-UAC CA Trust on Windows:** Automatically trusts locally generated certificates in the Windows User certificate store without asking for administrator privileges.
- 🔒 **Automatic NSS/Chrome trust on Linux:** Automatically registers certificates in Chromium/NSS stores if `certutil` is available.
- ⏱️ **Beautiful Block Page:** Renders a gorgeous glassmorphic dark-mode blocker page with a live circular progress bar and JavaScript countdown timer.
- 🔄 **Self-Cleaning Background Daemon:** Spawns a lightweight background controller that automatically disables proxy settings and kills the server once your focus session expires.

## Installation

Rerono is package-ready and can be installed locally with `uv` (recommended) or `pip`:

```bash
# Clone the repository and install it in editable/local mode:
uv pip install -e .

# Or install it globally using uv:
uv tool install .
```

## Usage

Rerono is controlled entirely through the terminal using a single command:

### 1. Predefined categories
Check what lists are defined in your configuration:
```bash
rerono list
```

### 2. Start a block session
To start blocking the `default` list for 30 minutes:
```bash
rerono start -d 30
```

To block specific categories or custom URLs for 45 minutes:
```bash
rerono start social reddit.com youtube.com/shorts -d 45
```

To block indefinitely (until stopped manually):
```bash
rerono start social
```

### 3. Check status
See details about the current active session, the rules being blocked, and the remaining time:
```bash
rerono status
```

### 4. Stop blocking
Gracefully terminate the session and restore your system proxy settings:
```bash
rerono stop
```

### 5. Emergency clean-up
If Rerono exits unexpectedly or you encounter connectivity issues, run the clean-up command to restore settings to normal and delete stale files:
```bash
rerono clean
```

## Configuration

Rerono configures itself on the first run. The configuration file `rules.yaml` defines your custom named categories.

- **Linux Path:** `~/.config/rerono/rules.yaml` (or XDG config directory)
- **Windows Path:** `%USERPROFILE%\.rerono\rules.yaml`

Example `rules.yaml`:
```yaml
default:
  - youtube.com/shorts
  - instagram.com/reels
  - facebook.com/watch
  - tiktok.com

social:
  - facebook.com
  - instagram.com
  - twitter.com
  - x.com

productivity:
  - reddit.com
  - youtube.com
  - twitch.tv
```

## HTTPS Interception Instructions

For path-specific blocking to work on secure HTTPS sites, your browser or system needs to trust the local certificate generated by `mitmproxy`:

- **Windows:** Automatically trusted during the first `rerono start` command.
- **Linux:** Automatically registered for Chrome/Chromium if the `certutil` CLI (from `libnss3-tools`) is installed.
  - For Firefox or system-wide trust, `rerono start` will print the specific manual commands to run, e.g.:
    ```bash
    sudo cp ~/.mitmproxy/mitmproxy-ca-cert.pem /usr/local/share/ca-certificates/rerono-ca.crt
    sudo update-ca-certificates
    ```
